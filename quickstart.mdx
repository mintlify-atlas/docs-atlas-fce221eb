<Note>
  You will need active API keys for **Azure AI Foundry** (GPT-4.1), **Google AI** (Gemini — for final synthesis), and **Exa AI** (web search) before Apollo can perform research. All three are required for the full research pipeline to work.
</Note>

## Prerequisites

- [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0)
- [Bun](https://bun.sh) (or Node.js 18+) for the frontend
- [PostgreSQL 15+](https://www.postgresql.org/download/) with the `pgvector` extension
- API keys: Azure AI Foundry, Google AI (Gemini), Exa AI

## Setup steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/manasseh-zw/apollo.git
    cd apollo
    ```
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the `Apollo.Api/` directory based on the example:

    ```bash
    cp Apollo.Api/.env.example Apollo.Api/.env
    ```

    Then fill in your values:

    ```env Apollo.Api/.env
    # Database — main application data
    DATABASE_URL=Host=localhost;Port=5432;Database=apollo;Username=postgres;Password=yourpassword

    # Database — Kernel Memory vector store (can be the same host, different DB)
    VECTOR_DB_URL=Host=localhost;Port=5432;Database=apollo_memory;Username=postgres;Password=yourpassword

    # JWT authentication
    JWT_SECRET_KEY=your-long-random-secret-key-at-least-32-chars
    JWT_ISSUER=apollo
    JWT_AUDIENCE=apollo_api

    # Azure AI Foundry — primary LLM for all three agents
    AZURE_AI_API_KEY=your-azure-ai-foundry-api-key
    AZURE_AI_ENDPOINT=https://your-resource.openai.azure.com/

    # Google Gemini — used for final report synthesis
    GOOGLE_AI_API_KEY=your-google-ai-api-key

    # Exa AI — web search and content retrieval
    EXA_API_KEY=your-exa-api-key

    # Frontend URL — used for CORS policy
    CLIENT_URL=http://localhost:5173
    ```

    <Tip>
      For easier deployment, use Docker instead of running the backend directly. See the [deployment guide](/deployment) for full instructions.
    </Tip>
  </Step>

  <Step title="Set up PostgreSQL with pgvector">
    Apollo requires two PostgreSQL databases: one for application data and one for Kernel Memory's vector store.

    ```sql
    -- Run these in psql or your PostgreSQL client
    CREATE DATABASE apollo;
    CREATE DATABASE apollo_memory;

    -- Enable pgvector in both databases
    \c apollo
    CREATE EXTENSION IF NOT EXISTS vector;

    \c apollo_memory
    CREATE EXTENSION IF NOT EXISTS vector;
    ```

    Then run the Entity Framework migrations to create the application schema:

    ```bash
    dotnet ef database update --project Apollo.Data/Apollo.Data.csproj --startup-project Apollo.Api/Apollo.Api.csproj
    ```
  </Step>

  <Step title="Run the backend API">
    ```bash
    dotnet run --project Apollo.Api/Apollo.Api.csproj
    ```

    The API starts on `https://localhost:5001` (or `http://localhost:5000`). You should see log output confirming database connection and service startup.

    In development mode, the Scalar API reference is available at `https://localhost:5001/scalar/v1`.
  </Step>

  <Step title="Run the frontend">
    The frontend lives in `Apollo.Client/`. Create a `.env` file there:

    ```env Apollo.Client/.env
    VITE_SERVER_URL=http://localhost:5000
    VITE_GOOGLE_CLIENT_ID=your-google-oauth-client-id
    ```

    Then install dependencies and start the dev server:

    <CodeGroup>
      ```bash Bun
      cd Apollo.Client
      bun install
      bun start
      ```

      ```bash npm
      cd Apollo.Client
      npm install
      npm run dev
      ```
    </CodeGroup>

    The frontend opens at `http://localhost:5173`.
  </Step>

  <Step title="Submit your first research query">
    1. Open `http://localhost:5173` in your browser
    2. Create an account or sign in
    3. Enter a research question — for example: *"What are the latest advances in quantum error correction?"*
    4. Click **Start Research**

    Apollo will begin planning the research, then you will see real-time progress updates as the agents work: web searching, ingesting content into the vector store, analyzing knowledge gaps, and finally synthesizing a comprehensive report.

    Research typically takes 3–10 minutes depending on the complexity of the topic and the number of knowledge gap cycles.
  </Step>
</Steps>

## What happens next

After submitting a query, the research pipeline runs fully in the background:

1. The **Research Planner** generates core research questions
2. The **.NET channels event bus** fires a research event
3. **Athena** searches the web using Exa AI and ingests results into pgvector
4. **Hermes** performs self-reflective RAG to identify gaps and fill them
5. The **Report Generator** synthesizes a final report using a large-context LLM

You can monitor progress in real-time via the frontend dashboard, or reconnect later — all updates are cached and replayed on reconnect.
