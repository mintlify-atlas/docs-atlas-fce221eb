title: "Research history"
description: "Retrieve a paginated list of the authenticated user's research sessions, ordered by most recently started."
---

<Badge color="green">GET</Badge> `/api/research/history`

Returns a paginated summary of all research sessions owned by the authenticated user. Each item contains only the session ID, title, and start time — use [Get research](/api-reference/research/get) to load the full plan and report for a specific session.

## Authentication

Required. The request must include the `apollo_access_token` HTTP-only cookie.

---

## Query parameters

<ParamField query="page" type="number" default="1">
  The page number to retrieve. Pages are 1-indexed. Defaults to `1`.
</ParamField>

<ParamField query="pageSize" type="number" default="5">
  The maximum number of items to return per page. Defaults to `5`.
</ParamField>

---

## Response

Returns a `PaginatedResponse<ResearchHistoryItemResponse>` object on success (`200 OK`).

<ResponseField name="items" type="object[]" required>
  The list of research history items for the requested page.
  <Expandable title="item properties">
    <ResponseField name="id" type="string" required>
      UUID of the research session.
    </ResponseField>
    <ResponseField name="title" type="string" required>
      Short label for the research session.
    </ResponseField>
    <ResponseField name="startedAt" type="string" required>
      ISO 8601 UTC timestamp of when the session was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="totalCount" type="number" required>
  Total number of research sessions across all pages.
</ResponseField>

<ResponseField name="page" type="number" required>
  The current page number (matches the `page` query parameter).
</ResponseField>

<ResponseField name="pageSize" type="number" required>
  The number of items per page (matches the `pageSize` query parameter).
</ResponseField>

<ResponseField name="hasMore" type="boolean" required>
  `true` when there are additional pages of results beyond the current one. Use this to drive "load more" or infinite scroll UIs.
</ResponseField>

---

## Error responses

| Status | When it occurs |
|--------|----------------|
| `401 Unauthorized` | No valid session cookie was sent, or the session has expired. |

## Code examples

<CodeGroup>

```bash cURL
# First page, default page size
curl --request GET \
  --url 'https://your-apollo-instance.com/api/research/history' \
  --cookie 'apollo_access_token=<your-token>'

# Second page with 10 items per page
curl --request GET \
  --url 'https://your-apollo-instance.com/api/research/history?page=2&pageSize=10' \
  --cookie 'apollo_access_token=<your-token>'
```

```typescript TypeScript
async function fetchAllHistory() {
  const allItems = [];
  let page = 1;
  let hasMore = true;

  while (hasMore) {
    const response = await fetch(
      `/api/research/history?page=${page}&pageSize=10`,
      { method: 'GET', credentials: 'include' }
    );

    if (!response.ok) {
      throw new Error(`Unexpected status ${response.status}`);
    }

    const data = await response.json();
    allItems.push(...data.items);
    hasMore = data.hasMore;
    page++;
  }

  return allItems;
}
```

</CodeGroup>
