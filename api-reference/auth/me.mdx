title: "Get current user"
description: "Return the profile of the currently authenticated user, identified by the session cookie."

<Badge color="green">GET</Badge> `/api/auth/me`

Returns the profile of the authenticated user whose identity is encoded in the `apollo_access_token` session cookie. Use this endpoint to verify a session is still valid or to refresh local user state.

## Authentication

Required. The request must include the `apollo_access_token` HTTP-only cookie issued by the [sign-up](/api-reference/auth/signup), [sign-in](/api-reference/auth/signin), or [Google sign-in](/api-reference/auth/google-signup) endpoints.

<Note>
  Browsers send this cookie automatically when `credentials: 'include'` is set on the fetch call. The cookie is `HttpOnly`, `Secure`, and `SameSite=None`, so it cannot be read from JavaScript — it is only transmitted by the browser on same-origin or cross-origin requests that include credentials.
</Note>

---

## Response

Returns a `UserDataResponse` object on success (`200 OK`).

<ResponseField name="id" type="string" required>
  UUID (version 7) uniquely identifying the user.
</ResponseField>

<ResponseField name="email" type="string" required>
  Email address associated with the account.
</ResponseField>

<ResponseField name="username" type="string">
  Display name for the user. May be `null` if not set.
</ResponseField>

<ResponseField name="avatarUrl" type="string">
  URL of the user's avatar image. Empty string if none has been set.
</ResponseField>

<ResponseField name="isEmailConfirmed" type="boolean" required>
  Whether the user has verified their email address.
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the account was created (UTC).
</ResponseField>

<ResponseField name="authProvider" type="string" required>
  Authentication method for the account: `"Email"` or `"Google"`.
</ResponseField>

---

## Error responses

| Status | When it occurs |
|--------|----------------|
| `401 Unauthorized` | No valid session cookie was sent, or the JWT in the cookie is invalid or expired. |
| `404 Not Found` | The user ID encoded in the JWT no longer exists in the database. |

---

## Code examples

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-apollo-instance.com/api/auth/me \
  --cookie 'apollo_access_token=<your-token>'
```

```typescript TypeScript
const response = await fetch('/api/auth/me', {
  method: 'GET',
  credentials: 'include', // sends the apollo_access_token cookie automatically
});

if (response.status === 401) {
  // Session expired or not authenticated — redirect to sign-in
  window.location.href = '/signin';
} else if (response.status === 404) {
  // Account no longer exists
  console.error('User not found');
} else {
  const user = await response.json();
  console.log(user.id, user.email, user.authProvider);
}
```

</CodeGroup>
